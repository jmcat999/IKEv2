# Proxy ARP 模式

高级模式，让 VPN 客户端获得和局域网同网段的 IP。

## 特点

- VPN 客户端使用局域网同网段地址，例如 `192.168.0.240/28` 或 `192.168.0.180-192.168.0.200`。
- 不做 MASQUERADE/NAT。
- 宿主机开启 Proxy ARP，让局域网设备可以找到 VPN 客户端 IP。
- 需要避开主路由 DHCP 地址池，避免 IP 冲突。

## 最简 `.env`

通常只需要改这两项：

```env
VPN_MODE=proxyarp
PROXYARP_VPN_POOL=192.168.0.240/28
```

也支持范围写法：

```env
VPN_MODE=proxyarp
PROXYARP_VPN_POOL=192.168.0.180-192.168.0.200
```

这段地址必须在路由器 DHCP 里排除，不能和已有设备冲突。

## DNS

默认情况下，如果不填写 `PROXYARP_DNS1`，脚本会从 `PROXYARP_VPN_POOL` 自动推导网关 DNS：

```text
192.168.0.240/28 -> 192.168.0.1
192.168.0.180-192.168.0.200 -> 192.168.0.1
```

如果你的路由器/DNS 不是 `.1`，请手动填写：

```env
PROXYARP_DNS1=192.168.0.254
PROXYARP_DNS2=1.1.1.1
```

## PROXYARP_LAN_SUBNET 是什么

一句话：

```text
PROXYARP_LAN_SUBNET = 服务端知道你的家里内网范围
```

它是给服务端脚本看的，主要用于防火墙和 Proxy ARP 判断。

比如你家里是：

```text
路由器：192.168.0.1
FnNas：192.168.0.2
其他设备：192.168.0.x
```

那你的家里内网范围通常就是：

```env
PROXYARP_LAN_SUBNET=192.168.0.0/24
```

默认会自动从 `PROXYARP_VPN_POOL` 推导：

```text
192.168.0.240/28 -> 192.168.0.0/24
192.168.0.180-192.168.0.200 -> 192.168.0.0/24
192.168.31.240/28 -> 192.168.31.0/24
```

一般不需要填写。

只有你的局域网不是 `/24`，或者自动推导不对，才需要手动写。

## PROXYARP_LOCAL_TS 是什么

一句话：

```text
PROXYARP_LOCAL_TS = 告诉客户端哪些目标地址走 VPN
```

它是给 VPN 客户端看的，决定客户端访问哪些地址时进入 VPN 隧道。

默认等于 `PROXYARP_LAN_SUBNET`。

比如：

```text
PROXYARP_LAN_SUBNET=192.168.0.0/24
PROXYARP_LOCAL_TS 默认就是 192.168.0.0/24
```

效果就是：

```text
客户端访问 192.168.0.x 走 VPN
客户端访问公网网站不走 VPN
```

一般不需要改。

如果你想让客户端所有 IPv4 流量都走 VPN，可以手动设置：

```env
PROXYARP_LOCAL_TS=0.0.0.0/0
```

不过 Proxy ARP 模式一般建议保持默认，只让家里内网流量走 VPN。

## 安全要求

- 不要使用整个 `192.168.0.0/24` 作为 VPN_POOL。
- 推荐只留小段，例如 `192.168.0.240/28` 或 `192.168.0.180-192.168.0.200`。
- 必须在主路由 DHCP 里排除这段地址。
- 如果客户端所在地网络也是 `192.168.0.0/24`，可能发生路由冲突。

## 适合场景

- 需要 VPN 客户端在内网里显示为同网段 IP。
- 需要某些只接受同网段访问的服务。
- 能够控制主路由 DHCP 地址池。
